## [title]
Ensure RBAC is enabled

## [profile]
Level 1 - Server

## [assessment status]
Automated

## [status]
draft

## [description]
RBAC allows granular control over permissions for users.

## [rationale statement]
RBAC is vital to ensure that the principle of least privilege is adhered to.

## [impact statement]

## [audit procedure]
Run the following command and verify that RBAC is active and the client is authenticated with an appropriate role:

```
talosctl config info
```
Expected output should include a `Roles` field, for example:

```
Roles:    os:admin
```

## [remediation procedure]
RBAC is permanently enabled in Talos Linux and cannot be disabled. Talos uses certificates to control access, each `talosconfig` file contains a certificate scoped to a specific role. To ensure users are operating with the least privileged role required, generate scoped client configuration files using the `--roles` flag:

```
talosctl config new --roles=os:admin admin
```
This above command creates a new `talosconfig` file named `admin` with a certificate granting `os:admin` access. To generate a configuration for a more restricted role:

```
talosctl config new --roles=os:reader reader
```

This creates a new `talosconfig` file named `reader` with a certificate granting read-only access. The same pattern applies for the `os:operator` and `os:etcd:backup` roles.

## [default value]
RBAC is always enabled and locked to `true`. It cannot be disabled via machine configuration.
